# 表格功能手动验收清单

本文档提供表格功能的手动验收测试步骤，对应 `notion-table-feature-plan.md` 中的 R 组任务。

## 准备工作

1. 启动 Cditor 应用
2. 创建一个新文档或打开测试文档
3. 确保处于 light theme（如需测试 dark theme，切换主题后重复测试）
4. 对照原型 `doc/prototypes/notion-table-prototype.html` 检查视觉和交互细节

## R-001: 插入 2x2 表格默认样式接近 Notion

**测试步骤：**
1. 在文档中输入 `/table` 或使用 slash 命令插入表格
2. 观察默认插入的 2x2 表格

**验收标准：**
- ✅ 表格有圆角边框（8px 圆角）
- ✅ 边框颜色为柔和的灰色（#e9e9e7）
- ✅ 第一行为 header row，背景色为暖灰色（#f7f6f4）
- ✅ 单元格之间有清晰但不过重的分隔线
- ✅ 单元格内有合适的 padding（10px 左右，8px 上下）
- ✅ 整体视觉与 Notion 表格风格接近

**状态：** [ ]

## 综合测试场景

### 场景 1：完整的表格编辑流程
1. 创建一个 3x3 表格
2. 填充所有单元格内容（包括中文、英文、数字）
3. 调整某些列宽和行高
4. 合并 header row 的前两列
5. 设置不同单元格的对齐方式（左、中、右）
6. 使用 row/column menu 插入新行列
7. 删除某一行
8. Reorder 两列
9. 保存文档
10. 关闭并重新打开文档
11. 验证所有内容、样式、结构保持一致

### 场景 2：大表格性能测试
1. 创建一个较大的表格（例如 20x10）
2. 填充所有单元格内容
3. 快速连续输入（测试 typing latency）
4. 快速 resize column（测试 drag frame budget）
5. 观察界面是否流畅，无明显卡顿

### 场景 3：Undo/Redo 完整性
1. 对表格进行一系列操作：
   - 输入内容
   - Resize
   - Merge
   - Reorder
   - 设置对齐
2. 逐步 undo 每个操作
3. 验证每步 undo 后状态正确
4. 再逐步 redo
5. 验证最终状态与初始一致
